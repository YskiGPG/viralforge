# ViralForge —— 项目背景与方案

> 本文档是项目的完整背景，包括需求来源、技术决策、用户体验设计和实施计划。
> 用于：自己回顾、与队友对齐、给Claude做上下文。

## 0. 一句话定位

ViralForge 是一个**对话式 AI 视频策划助手**。用户用自然语言描述一个视频想法（比如"我想拍攀岩时唱歌"），系统通过 Anthropic Agent SDK + Skills 架构，结合 YouTube 与 Reddit 双数据源，产出一份完整的、有数据支撑的 YouTube Shorts 创作方案。

## 2. 需求演化（为什么是这个方案）

### 起点
最初的方向是"做一个类似小红书 / Instagram 的实时热点 + AI 内容生成工具"——帮内容创作者更快地抓住热点并产出内容。

### 几个关键转折

**转折 1：数据源不可行**
- 小红书：没有公开 API，反爬非常激进，账号封禁风险
- Instagram：2026 年官方 API 不再支持任意用户/话题的公开发现
- TikTok：Research API 仅限学术机构申请，数据有 24h-7d 延迟
- → 最终选择 **YouTube Data API v3**（10000 配额/天，免费）+ **Reddit API**（10000 请求/月，免费）

**转折 2：从"图文笔记"转向"视频策划"**
- 用户场景更清晰、更聚焦（"我想拍攀岩唱歌的视频"比"帮我写护肤笔记"更具体）
- 视频是当前流量主战场，演示效果好
- 聚焦 **YouTube Shorts** 一个平台

**转折 3：从"下拉菜单"转向"对话式输入"**
- 用户不应该被限制在预定义的选项里
- AI 原生产品应该让用户用自然语言表达
- 后续追问也应该是对话式的，不是"重新填表"

**转折 4：从 LangGraph 转向 Anthropic SDK + Skills**
- Anthropic 2025 年 10 月发布的 Skills 是更前沿、更优雅的架构
- 减少第三方依赖，学到的东西更通用
- Skills 可复用、可扩展、可演示——加新功能就是加一个 SKILL.md

### 核心设计决策

**1. Anthropic SDK，不是 LangGraph**
- 官方原生支持，少装一层依赖
- Skills + Tools 是 Anthropic 一等公民概念
- 调试更直接，学到的东西可迁移性更强

**2. Skills 作为可复用能力模块**
- 每个 Skill 是一个文件夹，包含一个 `SKILL.md`（Markdown 指令）+ 可选辅助脚本
- 加新功能 = 加一个 SKILL.md，主程序不动
- 期末演示可以现场加新 Skill 展示扩展性

**3. 自适应路由（这是研究亮点）**
- 复杂请求（"帮我策划一个视频"）→ 完整 Skills 流水线（20-40s，流式输出）
- 简单追问（"换个标题"）→ 单 Skill，2-3s
- 大多数 multi-agent demo 都"无脑跑全流程"，浪费 token 和时间

**4. 双数据源**
- **YouTube** = 硬数据（真实视频的标题、播放量、缩略图模式）
- **Reddit** = 软数据（真实用户对这类内容的讨论与反应）
- 单一数据源做不到的：既有数据支撑又有"人味"

### 已规划的 Skills

| Skill 名称 | 职责 |
|---|---|
| `trend_analyst` | 从一组视频数据中提炼爆款规律（标题套路、钩子类型、节奏信号） |
| `hook_writer` | 基于视频概念生成 15 秒开场钩子结构 |
| `title_crafter` | 用经过验证的公式生成 3 个标题变体 |
| `platform_stylist` | 按目标平台（YouTube Shorts）调整文案风格 |

### 已规划的 Tools

| Tool 名称 | 职责 |
|---|---|
| `youtube_search_tool` | 调用 YouTube Data API v3，搜相关视频 + 拉元数据 |
| `reddit_search_tool` | 调用 Reddit API（PRAW），搜相关 subreddit 讨论 |

## 4. 用户体验全流程

### Step 1：着陆页（极简）
一个聊天输入框 + 几个示例提示。

### Step 2：用户输入想法
> "我想拍攀岩时边攀边唱稻香"

### Step 3：AI 流式工作（看到 Agent 思考的过程）
不是 loading 圈在转，而是看到 trend_scout / community_listener / plan_generator 一个个上场，文字一行行蹦出来。

### Step 4：产出完整策划方案
- 核心定位（不是"会唱歌的攀岩者"，而是"用唱歌测试攀岩极限"）
- 3 个标题候选（挑战型 / 系列型 / 情绪型）
- 15 秒钩子分镜（0-3s / 3-8s / 8-15s）
- 拍摄建议（设备 / 必拍镜头 / 避免拍什么）
- 发布策略（平台 / 标签 / 时间）
- "为什么这样设计"——基于真实数据的解释

### Step 5：对话式追问
- "如果我拍不到自己的脸怎么办？"→ 3 种方案（0 元 / $30 / $200）
- "换一首英文歌呢？"→ 5 个适合声音破音的歌曲推荐
- 简单追问走快速路径，2-3 秒回答

## 6. 三周时间线

### Week 1：基础设施
- 申请 YouTube Data API v3 key（Google Cloud Console）
- 申请 Reddit API 凭证（OAuth app）
- 注册 Anthropic API 账号（$5 免费额度）
- 实现 `youtube_search_tool` 和 `reddit_search_tool`
- 写第一个 Skill：`trend_analyst`
- 单流程 CLI 原型

### Week 2：完整流水线
- 写剩余 Skills：`hook_writer`、`title_crafter`、`platform_stylist`
- 构建 Orchestrator 与自适应路由逻辑
- 用 Streamlit 搭带流式输出的 Chat UI
- 端到端可运行 demo

### Week 3：打磨 + 评估
- 招募 5-8 位真实创作者做用户测试
- 实现对照基线（同样功能但用单一长 prompt，不用 Skills 架构）
- 测量：响应延迟、输出质量、token 消耗
- 准备最终演示 + 书面报告
